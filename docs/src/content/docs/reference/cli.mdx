kerf follows git's split between **porcelain** (human-facing, may evolve) and **plumbing** (scriptable, stable contract). Each command is documented on a per-task page, linked from the tree below.

## Global flags

Accepted on every subcommand:

| Flag | Purpose |
|---|---|
| `-v`, `--verbose` | Increase log verbosity. Repeat (`-vv`) for more detail. |
| `--identity-file <PATH>` | age identity file used to unwrap the DEK on decrypt. Env: `KERF_AGE_KEY_FILE` / `SOPS_AGE_KEY_FILE`. |

## Command tree

### Porcelain

```
# Encrypt, decrypt & exec
kerf encrypt <file> [--output PATH | --in-place] [recipient flags] [--encrypted-regex REGEX] [--format FORMAT]
kerf decrypt <file> [--output PATH] [--format FORMAT]
kerf exec    <file> [--format FORMAT] -- <command> [args...]

# Edit & set values
kerf edit    <file> [--format FORMAT]
kerf set     <file> <path>          # new value read from stdin, never argv
kerf unset   <file> <path>

# Inspect
kerf view    <file> [--path DOTTED.PATH] [--format FORMAT]
kerf diff    <old> <new> [--show-values] [--format FORMAT]
kerf verify  <file> [--format FORMAT]

# Recipients & rotation
kerf init    --recipient <type:value>... [-o PATH] [--path-regex REGEX] [--encrypted-regex REGEX] [--mac-all]
kerf keygen  -o <PATH>
kerf keys    add|remove <file> [recipient flags]
kerf keys    list <file>
kerf rotate  <file> [--reason "<msg>"] [--format FORMAT]
```

### Plumbing (stable)

```
kerf recipients     <file>
kerf metadata       <file>
kerf mac --verify   <file>
kerf path-encrypted <file> <path>   # exit 0 if encrypted, 1 if not — quiet
```

- [Plumbing commands](/kerf/reference/commands/plumbing/)

## Recipients & environment

The recipient flags (`--age`, `--kms`, `--gcp-kms`, `--azure-kv`), their `<type:value>` forms for `kerf init`, and the `KERF_*` / `SOPS_*` environment variables that feed them are documented under [Key providers](/kerf/key-providers/overview/).

## Exit codes

Commands signal outcomes through [exit codes](/kerf/reference/exit-codes/) — a stable contract that scripts should switch on rather than parsing stderr.

## Planned commands

These appear in [SPEC.md](https://github.com/idestis/kerf/blob/main/SPEC.md) §7 but are not yet in the shipped binary:

- `kerf updatekeys <file>` — reconcile a file's recipients against `.kerf.yaml` (add missing, remove stale) without rotating the DEK.
- `kerf import-sops <file>` — one-time conversion of an existing SOPS file to the kerf format.
